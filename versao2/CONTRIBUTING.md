# Guia de Contribuicao

## Fluxo recomendado

1. Atualize a `main` local com `git pull`.
2. Crie uma branch por tarefa, por exemplo: `feat/aula01-ex01`.
3. Faça commits pequenos e descritivos.
4. Abra Pull Request para revisao.

## Padrao de commits

- `feat:` nova funcionalidade
- `fix:` correcao de bug
- `docs:` documentacao
- `refactor:` mudanca sem alterar comportamento

## Revisao

- Rodar scripts alterados antes de abrir PR.
- Conferir nomes de arquivos e saidas esperadas do enunciado.
